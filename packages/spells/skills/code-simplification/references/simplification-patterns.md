# Simplification Patterns

Simplification patterns help identify and improve complex code.

## Common Patterns

1. **Extract Function**: Break large functions into smaller ones
2. **Remove Duplication**: Consolidate repeated logic
3. **Inline Trivial**: Remove unnecessary abstractions
4. **Flatten Nesting**: Reduce conditional depth
5. **Rename for Clarity**: Use meaningful names
6. **Remove Dead Code**: Delete unused code
7. **Simplify Logic**: Replace complex conditions with clearer equivalents

## Process

1. Identify pattern in code
2. Plan refactoring
3. Extract tests if needed
4. Refactor
5. Verify tests still pass
6. Commit change
