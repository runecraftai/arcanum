# Review Axes

Code review uses multiple axes to assess quality comprehensively.

## Review Axes

1. **Correctness**: Does it work? Does it handle errors?
2. **Clarity**: Is code easy to understand?
3. **Consistency**: Does it follow project conventions?
4. **Performance**: Is it efficient? Any bottlenecks?
5. **Security**: Are there vulnerabilities?
6. **Testing**: Is behavior adequately tested?
7. **Documentation**: Is it documented?

## Review Checklist

For each axis, assess:
- Pass or fail?
- Severity if fail (blocker, major, minor)
- Suggested improvement
