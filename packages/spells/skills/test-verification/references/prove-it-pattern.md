# Prove It Pattern

The prove-it pattern ensures code correctness through strategic test coverage.

## Pattern Steps

1. **Understand**: What behavior must be proven?
2. **Plan Tests**: What test cases cover the behavior?
3. **Write Tests**: Create test cases first
4. **Implement**: Make tests pass
5. **Verify**: Run full test suite
6. **Refactor**: Simplify implementation

## Coverage Goals

- Happy path: primary workflow
- Edge cases: boundary conditions
- Error handling: failure modes
- Integration: component interactions
