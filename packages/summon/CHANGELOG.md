## 0.16.0

### Minor Changes

- feat(summon): install external tools directly via `summon tools install` instead of generating `/setup-*` slash commands

  Previously, summon's `install-commands` emitted 7 `/setup-*` slash commands (`/setup-graphify`, `/setup-markitdown`, `/setup-context7`, `/setup-exa`, `/setup-grep-app`, `/setup-dynamic-context-pruning`, `/setup-agents-md`) whose body asked the LLM agent to detect the OS and run the right install command. That delegated real work to a non-deterministic layer and was impossible to run in non-interactive shells.

  Now summon installs the same tools itself via a declarative registry (`src/commands/tools-registry.ts`), with OS detection (`src/commands/runtime.ts`), safe `opencode.json` merging (`src/commands/mcp-config.ts`), and a two-stage presence check (binary + MCP entry).

  TUI gains a top-level category menu: **Skills / Commands / Setup**. Each category is a self-contained sub-flow. The "Install slash commands" entry moved from the skills action menu into its own **Commands** category.

  ### Breaking changes

  - The 7 `setup-*` slash commands are no longer generated. If you previously had `setup-*.md` files in your `.opencode/commands/` (or equivalent for Claude Code / Cursor), they will not be regenerated by re-running `install-commands`, but they are no longer referenced from the registry. Delete them or keep them as read-only references to the original prompt bodies (now in `docs/setup-prompts/`).
  - The `CommandMapping.body` field and `isStandaloneCommand()` helper are removed from the public API.
  - `src/commands/prompts/` was moved to `docs/setup-prompts/`. The contents are unchanged but are no longer in the published npm bundle (they were never loadable as code anyway).

  ### New APIs

  - `runToolsInstall(options)` in `src/commands/tools-install.ts` — programmatic entrypoint with `toolNames`, `scope`, `dryRun`, `checkOnly`, `projectRoot`, `envValues` options.
  - `summon tools install [tools...] [--scope=global|local] [--dry-run] [--check] [--project-root=PATH]` — CLI.
  - `summon tools list [--scope=global|local] [--project-root=PATH]` — show install state.

## 0.14.0

### Minor Changes

- e2c2690: feat(summon): per-runtime local/global picker for `install-commands`

  `install-commands` now asks which project roots to target (multi-path) and,
  for each detected runtime (Claude Code, OpenCode, Cursor), which location to
  write to (`local` = the project, `global` = `$HOME`). Cursor stays
  project-only. Generates files for every (projectRoot, runtime, location) pair
  in a single run. The `CommandGenerator` interface now exposes
  `supportedLocations`, `detectLocal`, `detectGlobal`, and a
  `location`-parameterized `generate`.

  Also rewrites the README to drop the stale "Scaffold" status, document the
  method × scope orthogonality, and add copy-pasteable examples for installing
  both skills and slash commands.
